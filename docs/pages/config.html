ConfigSource 是配置数据的来源接口，可组合多个源，后加载的覆盖先加载的同名键。

占位符解析
${key:default} · 嵌套占位符 · 默认值

Photon Config 支持在字符串中解析 ${key:default} 占位符，与 Spring 的 @Value 占位符语法保持一致：

方法说明Spring 等价
resolve_placeholders(expr)解析 ${key:default} 占位符，支持嵌套与默认值@Value("${property:defaultValue}")
v
import photon.config mut env := config.new_environment() env.add_source(map_source) // Simple placeholder / 简单占位符 value := env.resolve_placeholders('${database.host}') // → 'localhost' // Placeholder with default / 带默认值的占位符 value := env.resolve_placeholders('${database.port:5432}') // → '5432' if database.port not set // Nested placeholders / 嵌套占位符 url := env.resolve_placeholders('jdbc://${db.host:localhost}:${db.port:5432}/${db.name:mydb}') // → 'jdbc://localhost:5432/mydb' // Spring equivalent: @Value("${database.host:localhost}")
🔗
结构体绑定
bind_to_struct · bind_configuration_properties · MapPropertySource

Photon Config 提供类型安全的配置绑定机制，将配置值自动映射到 struct 字段：

方法 / 类型说明Spring 等价
bind_to_struct[T](binder)绑定 @[value] 标注的字段@Value 字段注入
bind_configuration_properties[T](binder, prefix)按前缀绑定配置（@ConfigurationProperties 等价）@ConfigurationProperties(prefix)
MapPropertySource内存配置源，用于编程式配置MapPropertySource
v
import photon.config // 1. bind_to_struct[T]() — Bind @[value] annotated fields @[component] pub struct DatabaseConfig { @[value('database.host')] host string @[value('database.port')] port int @[value('database.name')] name string } mut binder := config.new_property_binder(cfg) mut db_config := config.bind_to_struct[DatabaseConfig](binder)! println(db_config.host) // 'localhost' // 2. bind_configuration_properties[T]() — Prefix-based binding // Spring equivalent: @ConfigurationProperties(prefix = "database") pub struct RedisConfig { host string port int db int } mut redis_config := config.bind_configuration_properties[RedisConfig](binder, 'redis')! // Binds redis.host, redis.port, redis.db to struct fields // 3. MapPropertySource — Programmatic configuration / 编程式配置源 mut map_source := config.new_map_property_source('overrides', { 'database.host': 'prod-db.example.com' 'database.port': '3306' }) cfg.add_source(map_source) // Spring equivalent: @ConfigurationProperties(prefix = "database") // Laravel equivalent: Config::set('database.host', 'prod-db.example.com')
🚀 生产实践
  • 敏感信息（密码、密钥）使用环境变量注入，不要写入配置文件
  • 环境变量覆盖文件配置（EnvSource 优先级最高）
  • 使用 @[value('key')] 绑定到 struct 字段，避免散落的魔术字符串
  • Profile 特定配置只覆盖差异项，减少重复配置
  • 使用 resolve_placeholders('${key:default}') 统一配置占位符语法，与 Spring 保持一致
  • 使用 bind_configuration_properties[T](prefix) 实现类型安全的配置绑定，替代散落的 get_or() 调用
  • 使用 MapPropertySource 在测试中注入覆盖配置
