# Photon Framework — AGENTS.md 开发规范宪法

## 一、框架使命

Photon 是一个面向商业化应用的 V 语言企业级框架，对标 Spring Boot，提供：
- 注解驱动的声明式编程模型
- 编译期依赖注入与代码生成（零运行时反射开销）
- 完整的中间件生态（缓存、日志、ORM、锁、连接池）
- 极低的心智成本与极高的生产力

**核心原则：编译期优于运行期，显式优于隐式，约定优于配置。**

---

## 二、模块架构规范

### 2.1 模块分层
```
photon/
├── core/       # 核心容器：DI、生命周期、注解引擎
├── config/     # 配置管理：多源配置、属性绑定
├── log/        # 日志：分级日志、结构化输出
├── cache/      # 缓存：内存/Redis 多后端抽象
├── orm/        # ORM：实体映射、仓库模式、关系、迁移
├── pool/       # 连接池：通用对象池、数据库连接池
├── lock/       # 锁：本地互斥锁、分布式锁
├── web/        # Web：veb 增强、注解路由、中间件链
└── example/    # 示例应用
```

### 2.2 模块设计原则
1. **单一职责**：每个模块只解决一个问题域
2. **接口抽象**：模块间通过 trait（接口）解耦，核心不依赖具体实现
3. **可选依赖**：用户按需引入模块，不强制全部加载
4. **向后兼容**：公开 API 的变更必须经过弃用周期

## 三、注解（Attribute）规范

### 3.1 注解命名
Photon 注解使用 V 语言的 `@[attr]` 语法，命名遵循：
- **组件标记**：`@[component]`, `@[service]`, `@[repository]`, `@[controller]`, `@[configuration]`
- **依赖注入**：`@[autowired]`, `@[value('key')]`, `@[qualifier('name')]`
- **Web 路由**：`@[get('/path')]`, `@[post('/path')]`, `@[put('/path')]`, `@[delete('/path')]`
- **横切关注点**：`@[cacheable]`, `@[transactional]`, `@[scheduled('cron')]`
- **生命周期**：`@[post_construct]`, `@[pre_destroy]`, `@[lazy]`, `@[scope('singleton'|'prototype')]`

### 3.2 注解处理
- 所有注解在**编译期**通过 `comptime $for` 扫描并生成代码
- 禁止运行时反射解析注解（V 语言设计哲学）
- 注解处理器必须输出可读的编译错误信息

## 四、依赖注入（DI）规范

### 4.1 Bean 定义
```v
@[component]
pub struct UserService {
	@[autowired]
	user_repo &UserRepository
}
```

### 4.2 作用域
- **Singleton**（默认）：全局唯一实例
- **Prototype**：每次注入创建新实例
- **Request**：每个 HTTP 请求一个实例（Web 模块）

### 4.3 注入方式
- **字段注入**：`@[autowired]` 标注字段（推荐，简洁）
- **构造器注入**：通过 `@[autowired]` 标注构造函数参数
- 禁止循环依赖（编译期检测并报错）

### 4.4 Bean 生命周期
```
实例化 → 属性填充 → @post_construct → 就绪 → @pre_destroy → 销毁
```

## 五、编码规范

### 5.1 V 语言风格
- 严格遵循 V 语言官方风格指南
- 使用 `snake_case` 命名文件和变量
- 使用 `PascalCase` 命名 struct 和 trait
- 公开符号必须标注 `pub`

### 5.2 错误处理
- 优先使用 V 的 `!` 错误传播（`or` 块处理）
- 框架内部错误使用自定义错误类型
- 用户可见错误信息中英文双语

### 5.3 文件组织
- 每个模块一个入口文件 `<module>.v`
- 大型模块按功能拆分为多个文件
- 测试文件命名为 `<feature>_test.v`

### 5.4 注释规范
```v
// 单行注释用于简要说明

/*
 * 多行注释用于详细文档
 * 描述参数、返回值、副作用
 */
```

## 六、性能与安全

### 6.1 性能要求
- 零运行时反射：所有 DI、路由匹配在编译期完成
- 内存池化：频繁分配的对象使用 sync.Pool
- 并发安全：所有共享状态使用适当的同步原语

### 6.2 安全要求
- SQL 注入防护：使用参数化查询
- XSS 防护：模板引擎默认转义
- CSRF 防护：Web 模块内置 CSRF Token
- 敏感信息脱敏：日志中自动脱敏密码、Token

## 七、版本与兼容性

### 7.1 语义化版本
- MAJOR：不兼容的 API 变更
- MINOR：向后兼容的功能新增
- PATCH：向后兼容的问题修复

### 7.2 V 语言版本
- 最低支持 V 0.4.x
- 目标支持 V 0.5.x（最新稳定版）

## 八、贡献指南

### 8.1 提交流程
1. Fork 仓库，创建 feature 分支
2. 编写代码并添加测试
3. 运行 `v test photon/...` 确保全部通过
4. 提交 PR，描述变更内容与动机

### 8.2 Code Review 要点
- 是否遵循本宪法规范
- 是否包含充分的单元测试
- 是否引入不必要的依赖
- API 设计是否符合"低心智成本"原则

---

## 九、商业化职责划分

### 9.1 模块成熟度
- **Alpha**：实验性，API 可能变更
- **Beta**：功能完整，接受反馈
- **GA**：生产就绪，长期支持

### 9.2 商业支持
- 框架核心（core, web, config）：永久开源免费
- 高级特性（分布式锁、集群支持）：开放核心，企业版可扩展
- 商业授权：提供技术支持、SLA 保障、安全补丁
