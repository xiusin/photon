CLI · 命令行

Symfony Console 风格 CLI 框架、命令注册与调度、ANSI 样式、进度条、代码生成器 — Laravel Artisan 的 V 语言实现。

⌨️
快速上手
注册命令 · 解析参数 · 执行
v
import photon.cli // 1. 创建 CLI Application mut app := cli.new_application('myapp', '1.0.0') // 2. 添加内置命令 app.add_command(cli.new_list_command()) // list — 列出所有命令 app.add_command(cli.new_help_command()) // help — 显示命令帮助 app.add_command(cli.new_serve_command()) // serve — 启动开发服务器 app.add_command(cli.new_schedule_run_command()) // schedule:run app.add_command(cli.new_queue_work_command()) // queue:work // 3. 添加自定义命令 app.add_command(my_greet_command) // 4. 运行（自动解析 os.args） app.run() or { panic(err) } // Laravel 等价：php artisan // Spring 等价：Spring Shell

自定义命令
Command 接口 · 参数 · 选项
v
// 定义自定义命令 struct GreetCommand {} pub fn (c &GreetCommand) name() string { return 'greet' } pub fn (c &GreetCommand) description() string { return 'Greet a user' } pub fn (mut c GreetCommand) execute(mut ctx cli.CommandContext) ! { // 获取参数 name := ctx.argument('name') or { 'World' } // 获取选项 loud := ctx.option('loud') != '' msg := 'Hello, ${name}!' if loud { ctx.info(msg.to_upper()) } else { ctx.info(msg) } } // 使用方式： // myapp greet Alice --loud

样式化输出
ANSI 颜色 · 表格 · 进度条
v
// 颜色输出 ctx.info('Information message') // 绿色 ctx.warn('Warning message') // 黄色 ctx.error('Error message') // 红色 ctx.success('Operation completed!') // 绿色加粗 // 进度条 mut bar := ctx.new_progress_bar(100) for i in 0 .. 100 { do_work(i) bar.advance(1) } bar.finish() // 交互式输入 name := ctx.ask('What is your name?') yes := ctx.confirm('Continue?') // true/false secret := ctx.secret('Enter password:') // 隐藏输入 choice := ctx.choice('Select env:', ['dev', 'staging', 'prod']) // Laravel 等价：$this->info() / $this->confirm() / $this->choice()

代码生成器
make:command · make:controller · make:middleware
bash
# 内置代码生成命令 myapp make:command SendEmails # 生成命令模板 myapp make:controller UserController # 生成控制器模板 myapp make:middleware AuthMiddleware # 生成中间件模板 myapp make:repository UserRepository # 生成 Repository 模板 myapp make:service UserService # 生成 Service 模板 # Laravel 等价：php artisan make:controller
🚀 生产实践
  • 使用 make:* 命令生成脚手架，保持代码结构一致
  • 自定义命令遵循 namespace:action 命名规范（如 user:deactivate）
  • 长时间运行的任务使用进度条提供反馈
  • 敏感输入（密码）使用 secret() 而非 ask()
