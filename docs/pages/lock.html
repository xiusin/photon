本地互斥锁
LocalMutex · 指数退避自旋

基于 V 的 sync.Mutex 实现，使用指数退避策略（100μs → 50ms cap）平衡低延迟与低 CPU 消耗。

v
import photon.locking // 创建互斥锁 mut m := locking.new_mutex() // 阻塞获取锁 m.lock() // ... 临界区代码 m.unlock() // 非阻塞尝试获取锁 if m.try_lock() { // 获取成功 // ... 临界区代码 m.unlock() } else { // 获取失败，执行备选逻辑 } // Go 等价：sync.Mutex // Java 等价：ReentrantLock

DistributedLock · Redis / etcd 可插拔后端

// DistributedLock 接口 pub interface DistributedLock { acquire(key string, ttl_ms int) !bool release(key string) !bool renew(key string, ttl_ms int) !bool // 自动续期（看门狗） is_locked(key string) bool } // 使用 Redis 分布式锁 mut dlock := new_redis_distributed_lock(redis_client) // 获取锁（TTL 10 秒） locked := dlock.acquire('order:123', 10000)! if locked { // ... 处理业务 dlock.release('order:123')! } // 自动续期（防止业务执行时间超过 TTL） dlock.renew('order:123', 10000)! // Java 等价：Redisson RLock // Go 等价：go-redsync

读写锁
RWMutex · 读多写少场景优化
v
mut rw := locking.new_rwmutex() // 读锁（多个读者可并行） rw.rlock() data := read_data() rw.runlock() // 写锁（独占） rw.lock() write_data(new_value) rw.unlock()
🚀 生产实践
  • 分布式锁必须设置 TTL，防止死锁（进程崩溃后锁自动释放）
  • 长事务使用 renew() 续期，防止业务未完成锁已过期
  • 获取锁后使用 defer { lock.release() } 确保异常时释放
  • 本地锁适用于单进程内的资源保护，跨进程/跨节点使用分布式锁
  • 锁的粒度越细越好：order:123 优于 orders
