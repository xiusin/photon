veb.auth 认证桥接
密码哈希 · Token 生成与验证 · 安全随机数

桥接 veb.auth 的认证能力，提供 SHA256+salt 密码哈希、安全 Token 生成与验证、Token 管理器（存储/吊销/过期清理）等功能。与 Photon Security 模块无缝集成。

密码哈希与验证

v
import photon.security // 1. 创建密码编码器 encoder := security.new_password_encoder() // 2. 哈希密码（自动生成随机 salt） hashed := encoder.hash_password('my-secret-password')! // 返回格式: "sha256:salt_hex:hash_hex" // 3. 验证密码 ok := encoder.verify_password('my-secret-password', hashed)! println(ok) // true // 4. 检查密码是否需要重哈希 // （例如算法升级后，旧哈希需要更新） if encoder.needs_rehash(hashed) { new_hashed := encoder.hash_password(raw_password)! // 存储 new_hashed... } // Spring 等价: PasswordEncoder (BCryptPasswordEncoder) // Laravel 等价: Hash::make() / Hash::check()

// 生成安全随机 Token token := security.generate_token(32) // 32 字节随机数 → hex 字符串 println(token) // "a1b2c3d4e5f6..."（64 字符） // 验证 Token（防止时序攻击，使用恒定时间比较） valid := security.verify_token(submitted_token, stored_token) if !valid { return error('invalid token') }

Token 管理器（存储 / 吊销 / 过期清理）

TokenManager 提供 Token 的完整生命周期管理：生成、存储、验证、吊销、过期清理。适用于 API Token、密码重置链接、邮箱验证等场景。

v
// 1. 创建 Token 管理器 mut tm := security.new_token_manager(86400) // 24h 过期 // 2. 生成 Token 并关联用户 token := tm.generate('user-42')! println(token) // "f3e2d1c0..." // 3. 验证 Token user_id := tm.validate(token) or { // Token 不存在或已过期 return error('invalid or expired token') } println(user_id) // "user-42" // 4. 吊销 Token（登出 / 注销） tm.revoke(token)! // 5. 吊销用户的所有 Token tm.revoke_all_for_user('user-42')! // 6. 清理过期 Token（定期调用） cleaned := tm.cleanup_expired() println('cleaned ${cleaned} expired tokens')
💡 密码哈希说明：当前实现使用 SHA256+salt 方案。生产环境如需更强的安全性（抗 GPU/ASIC 暴力破解），建议升级到 argon2id 或 bcrypt。V 语言的 v.crypto.argon2 模块提供了 argon2id 支持。

OAuth 令牌交换桥接
veb.oauth → Photon — GitHub / Google / 微信等第三方登录

桥接 veb.oauth 的 OAuth 2.0 授权码流程（Authorization Code Flow），支持 GitHub、Google、微信等第三方登录。自动处理 state 参数防 CSRF、PKCE 校验。

配置 OAuth Provider

v
import photon.security // 1. 配置 GitHub OAuth github_config := security.OAuthConfig{ provider: 'github' client_id: 'your-github-client-id' client_secret: 'your-github-client-secret' redirect_uri: 'https://app.com/auth/github/callback' scopes: ['user:email', 'read:user'] auth_url: 'https://github.com/login/oauth/authorize' token_url: 'https://github.com/login/oauth/access_token' user_info_url: 'https://api.github.com/user' } // 2. 配置 Google OAuth google_config := security.OAuthConfig{ provider: 'google' client_id: 'your-google-client-id.apps.googleusercontent.com' client_secret: 'your-google-client-secret' redirect_uri: 'https://app.com/auth/google/callback' scopes: ['openid', 'email', 'profile'] auth_url: 'https://accounts.google.com/o/oauth2/v2/auth' token_url: 'https://oauth2.googleapis.com/token' user_info_url: 'https://www.googleapis.com/oauth2/v2/userinfo' } // 3. 创建 OAuth 桥接器 oauth := security.new_oauth_bridge(github_config)

完整 OAuth 登录流程

v
// ─── 步骤 1: 重定向到 OAuth Provider ─── @[get('/auth/github')] pub fn (mut ctx Context) github_login() veb.Result { // 生成授权 URL（含 state 防 CSRF） auth_url := oauth.get_authorization_url()! return ctx.redirect(auth_url, .found) } // ─── 步骤 2: OAuth Provider 回调 ─── @[get('/auth/github/callback')] pub fn (mut ctx Context) github_callback() veb.Result { code := ctx.query('code') or { return ctx.text('missing code') } state := ctx.query('state') or { return ctx.text('missing state') } // 1. 验证 state（防 CSRF） oauth.validate_state(state) or { return ctx.text('invalid state') } // 2. 用授权码交换 Access Token token_resp := oauth.exchange_code(code) or { return ctx.text('token exchange failed') } // 3. 获取用户信息 user_info := oauth.get_user_info(token_resp.access_token) or { return ctx.text('failed to get user info') } // 4. 在本地系统中创建/更新用户 user := user_service.find_or_create_oauth_user( provider: 'github' provider_id: user_info.id email: user_info.email name: user_info.name avatar: user_info.avatar_url )! // 5. 生成 JWT 并设置 Cookie jwt_token := jwt.create_token(user.id, user.roles)! ctx.set_cookie('access_token', jwt_token, ...) return ctx.redirect('/dashboard', .found) }

// 构建安全过滤链 mut chain := security.new_security_filter_chain() // 公开端点 chain.permit('/api/auth/login', ['POST']) chain.permit('/api/auth/register', ['POST']) chain.permit('/api/health', ['GET']) // 需要认证的端点 chain.require_auth('/api/users/**') chain.require_role('/api/admin/**', 'admin') // 拒绝所有其他请求 chain.deny_all() // Spring 等价：SecurityFilterChain bean // Laravel 等价：Route::middleware('auth:api')
🚀 生产实践
  • JWT Secret 使用 256 位以上的随机密钥，从环境变量注入
  • Access Token 有效期设置 15-60 分钟，Refresh Token 7 天
  • CSRF 保护在生产环境必须开启，API 场景可关闭（使用 JWT 替代）
  • RBAC 角色设计遵循最小权限原则
  • 密码使用 bcrypt/argon2 哈希，永远不要明文存储
  • 使用 TokenManager 管理 API Token，定期调用 cleanup_expired() 清理过期 Token
  • 用户登出时调用 revoke_all_for_user() 吊销所有 Token，确保安全
  • OAuth client_secret 从环境变量注入，state 参数必须验证
  • OAuth 回调处理中，使用 find_or_create 模式关联本地用户，避免重复注册
  • 密码重置链接使用 TokenManager.generate() 生成一次性 Token，设置较短过期时间（如 1 小时）
