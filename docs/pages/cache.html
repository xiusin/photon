CacheManager + 命名缓存
v
import photon.cache // 1. 创建 CacheManager mut cm := cache.new_cache_manager() // 2. 注册命名缓存 cm.register('users', cache.new_memory_cache('users')) cm.register('products', cache.new_memory_cache('products')) // 3. 基本操作 cm.set('user:42', '{"name":"Alice"}', 300)! // TTL 300 秒 val := cm.get('user:42')! // '{"name":"Alice"}' cm.has('user:42') // true cm.delete('user:42')! // 删除 cm.clear()! // 清空默认缓存 // 4. 使用命名缓存 users_cache := cm.get_cache('users') users_cache.set('list', '[...]', 60)! // Spring 等价：@Cacheable / CacheManager // Laravel 等价：Cache::put('key', 'value', 300)

Cache 接口
实现自定义缓存后端
v
// Cache 接口 — 所有缓存后端必须实现 pub interface Cache { mut: get(key string) !string set(key string, value string, ttl_seconds int) ! delete(key string) ! has(key string) bool clear() ! keys() []string size() int } // 自定义 Redis 后端示例 pub struct RedisCache { client &RedisClient prefix string } pub fn (mut rc RedisCache) get(key string) !string { return rc.client.get('${rc.prefix}:${key}') } pub fn (mut rc RedisCache) set(key string, value string, ttl_seconds int) ! { rc.client.setex('${rc.prefix}:${key}', ttl_seconds, value)! } // ... 实现其他方法

Singleflight 防击穿
合并并发请求 · 防止缓存雪崩

当多个协程同时请求同一个缓存键时，Singleflight 确保只有一个协程实际执行加载逻辑，其余协程共享结果。

v
// CacheManager 内置 Singleflight // 并发请求 user:42 时，只有一个执行数据库查询 result := cm.singleflight_execute('user:42', fn () !string { // 只有第一个请求执行此函数 user := db.find_user(42)! json_data := user.json() cm.set('user:42', json_data, 300)! return json_data })! // Go 等价：golang.org/x/sync/singleflight
✨
@[cacheable] 注解
声明式缓存 — 方法级缓存抽象
v
@[component] pub struct UserService { @[autowired] cache_manager &cache.CacheManager @[autowired] user_repo &UserRepository } @[cacheable] pub fn (s &UserService) get_user(id int) !User { // 首次调用走数据库，后续走缓存 return s.user_repo.find_by_id(id)! } // Spring 等价：@Cacheable(value = "users", key = "#id") // Laravel 等价：Cache::remember("user:$id", 300, fn() => User::find($id))
🚀 生产实践
  • 设置合理的 TTL，避免缓存永不过期导致数据不一致
  • 使用 Singleflight 防止缓存击穿（热点 key 过期瞬间大量请求打到数据库）
  • 生产环境使用 Redis 后端，开发环境使用 Memory 后端
  • 缓存键使用 namespace:key 格式，避免冲突
  • 对序列化数据使用 JSON 格式，便于调试和跨语言读取
