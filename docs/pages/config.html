占位符解析
${key:default} · 嵌套占位符 · 默认值

Photon Config 支持在字符串中解析 ${key:default} 占位符，与 Spring 的 @Value 占位符语法保持一致：

方法说明Spring 等价
resolve_placeholders(expr)解析 ${key:default} 占位符，支持嵌套与默认值@Value("${property:defaultValue}")
v
import photon.config mut env := config.new_environment() env.add_source(map_source) // Simple placeholder / 简单占位符 value := env.resolve_placeholders('${database.host}') // → 'localhost' // Placeholder with default / 带默认值的占位符 value := env.resolve_placeholders('${database.port:5432}') // → '5432' if database.port not set // Nested placeholders / 嵌套占位符 url := env.resolve_placeholders('jdbc://${db.host:localhost}:${db.port:5432}/${db.name:mydb}') // → 'jdbc://localhost:5432/mydb' // Spring equivalent: @Value("${database.host:localhost}")
🔗
结构体绑定
bind_to_struct · bind_configuration_properties · MapPropertySource

Photon Config 提供类型安全的配置绑定机制，将配置值自动映射到 struct 字段：

方法 / 类型说明Spring 等价
bind_to_struct[T](binder)绑定 @[value] 标注的字段@Value 字段注入
bind_configuration_properties[T](binder, prefix)按前缀绑定配置（@ConfigurationProperties 等价）@ConfigurationProperties(prefix)
MapPropertySource内存配置源，用于编程式配置MapPropertySource
v
import photon.config // 1. bind_to_struct[T]() — Bind @[value] annotated fields @[component] pub struct DatabaseConfig { @[value: 'database.host'] host string @[value: 'database.port'] port int @[value: 'database.name'] name string } mut binder := config.new_property_binder(cfg) mut db_config := config.bind_to_struct[DatabaseConfig](binder)! println(db_config.host) // 'localhost' // 2. bind_configuration_properties[T]() — Prefix-based binding // Spring equivalent: @ConfigurationProperties(prefix = "database") pub struct RedisConfig { host string port int db int } mut redis_config := config.bind_configuration_properties[RedisConfig](binder, 'redis')! // Binds redis.host, redis.port, redis.db to struct fields // 3. MapPropertySource — Programmatic configuration / 编程式配置源 mut map_source := config.new_map_property_source('overrides', { 'database.host': 'prod-db.example.com' 'database.port': '3306' }) cfg.add_source(map_source) // Spring equivalent: @ConfigurationProperties(prefix = "database") // Laravel equivalent: Config::set('database.host', 'prod-db.example.com')
🚀 生产实践
  • 敏感信息（密码、密钥）使用环境变量注入，不要写入配置文件
  • 环境变量覆盖文件配置（EnvSource 优先级最高）
  • 使用 @[value: 'key'] 绑定到 struct 字段，避免散落的魔术字符串
  • Profile 特定配置只覆盖差异项，减少重复配置
  • 使用 resolve_placeholders('${key:default}') 统一配置占位符语法，与 Spring 保持一致
  • 使用 bind_configuration_properties[T](prefix) 实现类型安全的配置绑定，替代散落的 get_or() 调用
  • 使用 MapPropertySource 在测试中注入覆盖配置

YAML / TOML 配置源
YamlConfigSource · TomlConfigSource — Spring Boot application.yml 等价

Photon 提供完整的 YAML 和 TOML 配置文件加载支持，实现 PropertySource 接口，可直接用于 Environment。

配置源说明Spring 等价

TomlConfigSourceTOML 配置文件，使用 V 官方 toml 模块（完整 TOML 1.0 规范）—

ConfigPropertySourceAdapter将 Config 适配为 PropertySource—

YAML 配置源

v
import photon.config // 创建 YAML 配置源 yaml_source := config.new_yaml_source('config/application.yml') // Profile 特定的 YAML 配置源 dev_source := config.new_profile_yaml_source('config/application.yml', 'dev') // → 自动构造路径 'config/application-dev.yml' // 添加到 Environment mut env := config.new_environment() env.add_source(yaml_source) env.add_source(dev_source) // YAML 文件示例： # app: # name: MyApp # port: 8080 # database: # host: localhost # port: 5432 # pool: # max: 20 // 读取扁平化后的 key（点号分隔） name := env.get_property('app.name') // 'MyApp' max_pool := env.get_property('database.pool.max') // '20'

TOML 配置源

v
import photon.config // 创建 TOML 配置源（使用 V 官方 toml 模块） toml_source := config.new_toml_source('config/app.toml') // Profile 特定的 TOML 配置源 prod_source := config.new_profile_toml_source('config/app.toml', 'prod') // → 自动构造路径 'config/app-prod.toml' env.add_source(toml_source) // TOML 文件示例： # [app] # name = "MyApp" # port = 8080 # # [database] # host = "localhost" # port = 5432 // 读取扁平化后的 key name := env.get_property('app.name') // 'MyApp'

Profile 感知批量加载

v
// 一次性加载基础配置 + Profile 覆盖配置 // 后加载的 Profile 配置覆盖基础配置中的同名键 // YAML 方式 props := config.load_yaml_with_profiles('config/application.yml', ['dev', 'local'])! // → 加载 application.yml → 覆盖 application-dev.yml → 覆盖 application-local.yml // TOML 方式 props2 := config.load_toml_with_profiles('config/app.toml', ['prod'])! // → 加载 app.toml → 覆盖 app-prod.toml // Spring 等价：application.yml + application-dev.yml + application-local.yml
⚠️ V 0.5.1 TOML 兼容性：V 0.5.1 的 C 后端在为 map[string]toml.Any 和 []toml.Any 等复杂嵌套泛型类型生成 _v_type_idx_ 函数时会产生无效 C 函数名。Photon 的 flatten_toml_any() 通过 to_toml() 字符串检测 + as_map()/array() 方法绕过此问题，无需等待编译器修复。

PropertySource 完整 API
统一接口 · 多源合并 · 缺失键返回 none

所有配置源实现统一的 PropertySource 接口，get_property() 在键不存在时返回 none（而非空字符串），确保配置缺失可被正确检测。

方法返回值说明
source.get_property(key)?string获取属性值，不存在返回 none
source.contains_property(key)bool检查属性是否存在
source.get_all_with_prefix(prefix)map[string]string获取指定前缀的所有属性
source.name()string配置源名称（用于调试）

MapPropertySource 编程式配置

v
// 创建内存配置源（适合测试覆盖、动态配置） mut overrides := config.new_map_property_source('overrides', { 'database.host': 'prod-db.example.com' 'database.port': '3306' 'app.debug': 'false' }) env.add_source(overrides) // 检查属性是否存在（none 语义） val := env.get_property('database.host') or { 'localhost' // 键不存在时的默认值 } // Spring 等价：MapPropertySource / @TestPropertySource // Laravel 等价：Config::set('database.host', 'prod-db.example.com')

Environment API
多源优先级链 · 属性查找 · 占位符解析
方法返回值说明
config.new_environment()&Environment创建环境实例
env.add_source(source)—添加 PropertySource（后添加优先级更高）
env.get_property(key)string按优先级链查找属性
env.get_property_or(key, default)string查找属性或返回默认值
env.has_property(key)bool检查属性是否存在
env.set_property(key, value)—设置编程式属性
env.resolve_placeholders(expr)string解析 ${key:default} 占位符
env.get_active_profiles()[]string获取激活的 Profile 列表
💡 优先级链：属性查找按源添加顺序的逆序进行 — 后添加的源优先级更高。典型优先级：CLI 参数 > 环境变量 > Profile 配置 > 基础配置 > 编程式默认值。
