## 二、P0 级缺陷 | 必须优先修复

### 2.1 `security/encryption.v` — XOR 密码依然存在

**文件**: `security/encryption.v`（37 行）
**问题**: XOR + hex 密码依然存在。`cipher.v`（300 行）已有 AES-256-CBC + HMAC，但 `encryption.v` 未标注废弃。

```v
pub struct Encrypter {  // ← 未标注 @[deprecated]
    cipher_variant CipherVariant
    key            string
}
```

**影响**: 用户可能误用 XOR 加密，导致安全性严重不足。

**修复方案**:
- [ ] 为 `Encrypter` 添加 `@[deprecated]` 属性
- [ ] 添加 `deprecated_encrypter` 包装，println 警告信息
- [ ] 在 `docs/encryption.md` 标注 XOR 为「仅演示用，生产请用 `cipher.AesCipher`」
- [ ] 后续版本移除 `encryption.v`

---

### 2.2 `security/hashing.v` — FNV-1a 不适用于密码哈希

**文件**: `security/hashing.v`（约 70 行）
**问题**: 使用 FNV-1a 非加密哈希，用于密码存储。FNV-1a 是快速非加密哈希，**不是** bcrypt/argon2。

```v
pub fn hash_password(password string) string {
    return fnv1a_hash(password)  // ← 这不是密码哈希，是可逆攻击的
}
```

**影响**: 密码可以被暴力破解，时间成本极低。**这是安全漏洞。**

**修复方案**:
- [ ] 引入 bcrypt（V 生态有 `x.bcrypt` 或纯 V 实现）
- [ ] 或引入 argon2（更现代的密码哈希算法）
- [ ] 当前 `hash_password` 改名为 `fnv1a_hash` 并标注不安全
- [ ] 添加密码哈希接口 `PasswordEncoder`（Strategy 模式，参考 Spring Security）：
```v
pub interface PasswordEncoder {
    encode(raw_password string) !string
    matches(raw_password string, encoded string) !bool
    upgrade_needed(encoded string) bool  // 检查是否需要升级哈希算法
}
pub struct BCryptPasswordEncoder { ... }
pub struct Argon2PasswordEncoder { ... }
```

### 2.3 `web/kernel.v` — `handle()` 方法不完整

**文件**: `web/kernel.v`（79 行）
**问题**: `handle()` 方法是一个半完成的骨架——分发事件但「Process request here」是占位注释。

```v
pub fn (mut k HttpKernel) handle() ! {
    k.dispatch(.request, unsafe { nil })
    k.dispatch(.controller, unsafe { nil })
    // Process request here    ← ❌ 占位符，没有任何实际逻辑
    k.dispatch(.response, unsafe { nil })
}
```

**影响**: HttpKernel 无法处理真实请求，web 模块直接绕过了 kernel 事件模型。
检查 `web/web.v` 发现 veb 替换了 HttpKernel 的角色。

**修复方案**:
- [ ] 完成 `handle()` 方法，或
- [ ] 添加 `@[deprecated]` 标注，说明已由 veb 接管
- [ ] 如果保留，需注入真实 HandlerResolver 到 `handle()` 中：
```v
pub fn (mut k HttpKernel) handle(ctx &veb.Context) ! {
    k.dispatch(.request, ctx)
    handler := k.resolve_handler(ctx) or {
        k.dispatch(.exception, err)
        return err
    }
    result := handler.execute(ctx) or {
        k.dispatch(.exception, err)
        return err
    }
    k.dispatch(.response, result)
    k.dispatch(.terminate, unsafe { nil })
}
```

### 2.4 `orm/transaction.v` — 事务管理是纯状态机

**文件**: `orm/transaction.v`（166 行）
**问题**: `TransactionManager` 的 `begin()`/`commit()`/`rollback()` 只跟踪布尔状态，**不接收数据库连接参数**。

```v
pub fn (mut tm TransactionManager) begin() ! {
    if tm.active { return error('transaction already active') }
    tm.active = true  // ← 只是设了个布尔值，没连接数据库
}
pub fn (mut tm TransactionManager) commit() ! {
    if !tm.active { return error('no active transaction') }
    tm.active = false  // ← 没有调用实际的数据库 commit
}
```

**影响**: `@[transactional]` 注解声明了 7 种传播行为，但实际运行中**没有任何事务效果**。

**修复方案**:
- [ ] 添加数据库连接/事务对象字段到 `TransactionManager`：
```v
pub struct TransactionManager[T] {
    db       &orm.DB          // 数据库连接
    tx       &orm.Tx          // 实际事务对象
    savepoints []orm.Savepoint // savepoint 链
    active   bool
    ...
}
fn (mut tm TransactionManager[T]) begin(db &orm.DB) ! {
    tm.tx = orm.begin(mut db)!
    tm.active = true
}
fn (mut tm TransactionManager[T]) commit() ! {
    tm.tx.commit()!
    tm.active = false
}
```
- [ ] 重新设计为非泛型 + 接口，避免 V 泛型限制
- [ ] `execute()` 方法需要接受数据库连接参数

## 三、P1 级缺口 | 重要架构缺失

### 3.1 密码哈希体系 — 无 PasswordEncoder 接口

**对标 Spring Security**: `PasswordEncoder` + `DelegatingPasswordEncoder`

**当前问题**: `hashing.v` 的 `hash_password` 是硬编码的 FNV-1a，不可插拔。

**优化方案**:
```v
pub interface PasswordEncoder {
    encode(raw_password string) !string
    matches(raw_password string, encoded string) !bool
}
pub struct BCryptPasswordEncoder { ... }
pub struct Argon2PasswordEncoder { ... }
// 迁移支持
pub struct DelegatingPasswordEncoder {
    default_encoder PasswordEncoder
    encoders map[string]PasswordEncoder  // idForEncode → encoder
}
```
参考: [Spring Security PasswordEncoder](https://docs.spring.io/spring-security/reference/features/authentication/password-storage.html)

---

### 3.2 缓存抽象 — 无 CacheManager 体系

**对标 Spring Cache**: `CacheManager` + `Cache` + `@Cacheable/@CacheEvict/@CachePut`

**当前问题**: `memory.v` 是硬编码实现，`cache/annotation.v` 只实现了 `@[cacheable]`。

**缺失项**:
- [ ] `CacheManager` 接口（可注册 Redis/EhCache/Local 后端）
- [ ] `@[cache_evict]` 注解
- [ ] `@[cache_put]` 注解
- [ ] `@[cache_config]` 类级配置（cacheNames, keyGenerator）

```v
pub interface CacheManager {
    get_cache(name string) !Cache
    get_cache_names() []string
}
pub interface Cache {
    get(key string) !ValueWrapper
    put(key string, value ValueWrapper) !
    evict(key string) !
    clear() !
}
```

---

### 3.3 消息/事件体系 — 缺少异步消息中间件抽象

**对标 Spring**: `ApplicationEventPublisher`（已有）+ JMS/RabbitMQ/Kafka 抽象（缺失）

**当前问题**: `core/event_bus.v` 实现了同步事件，但缺少：
- [ ] 异步事件（需要 go 程池）
- [ ] 事件监听器排序/优先级
- [ ] 事务事件监听器（`@[transactional_event_listener]`）
- [ ] 消息队列抽象（Queue → JMS/RabbitMQ/Kafka adapter）

---

### 3.4 类型转换 — 无 ConversionService

**对标 Spring**: `ConversionService` + `Converter`/`GenericConverter` + `Formatter`

**当前问题**: DTO 绑定在 `web/bind.v` 中硬编码类型转换，不可扩展。

**优化方案**:
```v
pub interface Converter[S, T] {
    convert(source S) !T
}
pub interface ConversionService {
    can_convert(source_type string, target_type string) bool
    convert[T](source voidptr) !T
    add_converter[S, T](converter Converter[S, T])
}
```
参考: [Spring Core ConversionService](https://docs.spring.io/spring-framework/reference/core/validation/convert.html)

---

### 3.5 内容协商 — 无 ContentNegotiation

**对标 Spring**: `ContentNegotiationManager` + `ContentNegotiationStrategy`

**当前问题**: web 模块没有内容协商。不支持根据 `Accept` 头返回 JSON/XML/HTML。

**优化方案**:
```v
pub struct ContentNegotiationManager {
    strategies []ContentNegotiationStrategy
}
pub interface ContentNegotiationStrategy {
    resolve_content_type(request voidptr) !string
}
// 内置策略: AcceptHeaderStrategy, ParameterStrategy, FixedStrategy
```

## 四、P2 级优化 | 代码质量与设计一致性

### 4.1 命名规范不一致

| 问题 | 示例 | 建议 |
|:-----|:------|:------|
| 混合函数前缀 | `new_*`（http/core/orm）vs `New*` 不存在 | 统一 `new_*`（已基本一致✅） |
| 模块内函数可见性 | 部分 `fn` 内部函数应该 `pub` 但没开放 | 每个模块添加 `pub use` 扫描 |
| 错误类型 | 散用 `error(string)`，缺少领域错误类型 | 添加 `PhotonError`+`ErrorCode` 枚举 |

### 4.2 指针使用过度

```v
// ❌ 过度使用 & 指针
pub struct TransactionManager {
    ...
}
pub fn new_transaction_manager() &TransactionManager { return &TransactionManager{} }

// ✅ V 值类型更安全（没有悬挂指针风险）
pub fn new_transaction_manager() TransactionManager { return TransactionManager{} }
pub fn (mut tm TransactionManager) begin() ! { ... }
```

**影响**: `&` 指针可能在并发场景产生悬挂引用。V 的值类型通过 `mut` 接收器同样可变更，更安全。

**建议**: 逐步检查，优先用值类型 + `mut` 接收器，除非有明确的共享需求。

### 4.3 `unsafe { nil }` 回调泛滥

大量使用 `unsafe { nil }` 作为函数指针默认值，类型安全为零。

```v
// ❌ unsafe { nil }
pub struct ClientHttpRequestInterceptor {
    intercept_fn fn (entity RequestEntity, next fn (RequestEntity) !ResponseEntity) !ResponseEntity = unsafe { nil }
}
```

**建议**: 用 Option 类型 `?fn(...)` 或建立 `NoopInterceptor` 哨兵对象。

### 4.4 缺少 benchmark 测试

只有 `locking/bench_test.v` 有 benchmark。Cache/ORM/Web 等核心模块缺少性能基准。

```
现有 bench: locking/bench_test.v
缺失 bench: cache/, orm/, web/, security/
```

**建议**: 每个 Beta 级模块至少 1 个 benchmark 测试。

### 4.5 错误处理风格不统一

| 模块 | 风格 | 问题 |
|:-----|:------|:------|
| http | `!ResponseEntity` | ✅ 一致 |
| core | `!voidptr` + `error(string)` | ✅ 一致 |
| security/cipher | `!string` + 领域错误 | ⚠️ 混合 `panic` 与 `return error` |
| orm/transaction | `!` 但不操作数据库 | ❌ 虚实现 |
| web/kernel | `!` 但不调用 veb | ❌ 半完成 |

**优化**: 对标 Spring `MessageSource`：

```v
pub interface MessageSource {

```

### 5.5 指标 / 健康检查（对标 Spring Actuator）

**缺失**: 无 `/actuator/health`、`/actuator/metrics` 等价物。

**优化**:
```v
pub interface HealthIndicator {
    health() !Health
}
pub struct Health { status: HealthStatus, details: map[string]string }
pub struct HealthAggregator {
    indicators   []HealthIndicator
    aggregate() !Health
}
```

### 5.6 协议扩展

**缺失**: 无 HTTPS/SSL 支持，无 HTTP/2 支持，无 HTTP 代理支持。

**当前**: 依赖 V 标准库 `net.http` 的能力。V 0.5.1 的 `net.http` 有 `use_http2` 配置。

**优化**: RestTemplate 暴露 SSL/代理配置接口：
```v
rt := new_rest_template()
rt.set_ssl_config(SSLConfig{enable: true, cert_file: 'client.crt'})
rt.set_proxy(ProxyConfig{host: 'proxy.com', port: 8080})
```

### 8.1 PasswordEncoder 接口（参考 Spring Security）

```v
// security/password_encoder.v
pub interface PasswordEncoder {
    encode(raw_password string) !string
    matches(raw_password string, encoded string) !bool
}

// BCrypt implementation
pub struct BCryptPasswordEncoder {
    strength int = 10  // cost factor
}
pub fn (e BCryptPasswordEncoder) encode(raw_password string) !string {
    // Use x.bcrypt or pure V implementation
    return error('TODO: implement bcrypt')
}
pub fn (e BCryptPasswordEncoder) matches(raw_password string, encoded string) !bool {
    return error('TODO: implement bcrypt')
}

// Legacy FNV-1a adaptor (for migration)
pub struct FnvPasswordEncoder {}
pub fn (e FnvPasswordEncoder) encode(raw_password string) !string {
    return fnv1a_hash(raw_password)
}
pub fn (e FnvPasswordEncoder) matches(raw_password string, encoded string) !bool {
    return fnv1a_hash(raw_password) == encoded
}

// DelegatingPasswordEncoder (migration helper)
pub struct DelegatingPasswordEncoder {
    default_encoder PasswordEncoder
    encoders map[string]PasswordEncoder  // id → encoder
}
pub fn (e DelegatingPasswordEncoder) encode(raw_password string) !string {
    encoded := e.default_encoder.encode(raw_password)!
    return '{${e.default_encoder_id()}}${encoded}'
}
pub fn (e DelegatingPasswordEncoder) matches(raw_password string, encoded string) !bool {
    id := encoded[1..encoded.index('}') or { return false }]
    encoder := e.encoders[id] or { return false }
    return encoder.matches(raw_password, encoded[encoded.index('}') or { return 0 } + 1..])?
}
```
