// 定义自定义命令 struct GreetCommand {} pub fn (c &GreetCommand) name() string { return 'greet' } pub fn (c &GreetCommand) description() string { return 'Greet a user' } pub fn (mut c GreetCommand) execute(mut ctx cli.CommandContext) ! { // 获取参数 name := ctx.argument('name') or { 'World' } // 获取选项 loud := ctx.option('loud') != '' msg := 'Hello, ${name}!' if loud { ctx.info(msg.to_upper()) } else { ctx.info(msg) } } // 使用方式： // myapp greet Alice --loud

命令注解
@[command] · @[option] · @[argument] · 编译期扫描

Photon CLI 提供注解驱动的命令定义，在编译期扫描命令元数据，实现声明式命令注册：

注解 / 方法说明Spring Shell 等价
@[command]标记 struct 为 CLI 命令@ShellComponent
@[command: 'name']指定命令名称@ShellMethod(value = "name")
@[description('help text')]命令描述@ShellMethod(help = "text")
@[option('name')]命令选项（--name）@ShellOption
@[argument]位置参数—
@[required]必填参数@ShellOption(defaultValue = "__NONE__")
@[default('value')]默认值@ShellOption(defaultValue = "value")
scan_command[T]()编译期扫描命令注解，返回 CommandInfo—
register_command_from[T]()从注解 struct 自动注册命令—
v
import photon.cli // Define command with annotations / 使用注解定义命令 @[command] @[description('Send notifications to users')] pub struct NotifyCommand { pub: @[option('channel')] @[default('email')] channel string @[argument] @[required] user_id string @[option('dry-run')] dry_run bool } // Comptime scan / 编译期扫描 info := cli.scan_command[NotifyCommand]() println(info.name) // 'NotifyCommand' println(info.description) // 'Send notifications to users' // Auto-register / 自动注册 mut app := cli.new_application('myapp', '1.0.0') cli.register_command_from[NotifyCommand](mut app)! // Spring Shell equivalent: @ShellMethod // Laravel equivalent: Artisan command signature

命令帮助生成
CommandHelp · AnnotatedCommand · bind_input · man page

Photon CLI 支持从注解自动生成帮助文本，并提供输入绑定与 man page 生成功能：

类型 / 方法说明Laravel 等价
CommandHelp结构化帮助文本构建器—
AnnotatedCommand带自动生成帮助的命令—
bind_input[T]()将 CLI 输入绑定到 struct 字段—
generate_help(app)生成所有命令的帮助文本php artisan help
generate_man_page(app)生成 man page 格式文档—
v
import photon.cli // 1. Auto-generated help from annotations / 从注解自动生成帮助 mut cmd := cli.AnnotatedCommand{ name: 'notify' description: 'Send notifications to users' } cmd.add_option('channel', 'Notification channel', 'email') cmd.add_argument('user_id', 'Target user ID', true) println(cmd.to_help_text()) // 2. bind_input[T]() — Bind input to struct / 绑定输入到结构体 mut input := cli.CommandContext{ args: {'user_id': '42'} options: {'channel': 'sms', 'dry-run': 'true'} } mut notify := cli.bind_input[NotifyCommand](mut input)! println(notify.user_id) // '42' println(notify.channel) // 'sms' // 3. Generate help for all commands / 生成所有命令的帮助 mut app := cli.new_application('myapp', '1.0.0') help_text := cli.generate_help(mut app) println(help_text) // 4. Generate man page / 生成 man page man_page := cli.generate_man_page(mut app) os.write_file('myapp.1', man_page) or {} // Laravel equivalent: php artisan help notify // Spring Shell equivalent: help command
🚀 生产实践
  • 使用 make:* 命令生成脚手架，保持代码结构一致
  • 自定义命令遵循 namespace:action 命名规范（如 user:deactivate）
  • 长时间运行的任务使用进度条提供反馈
  • 敏感输入（密码）使用 secret() 而非 ask()
  • 使用 @[command] + scan_command[T]() 实现编译期命令元数据扫描，避免手动配置
  • 使用 bind_input[T]() 实现类型安全的输入绑定，替代手动解析参数
  • 使用 generate_man_page() 为生产环境生成标准 man page 文档
