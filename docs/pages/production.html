安全加固
安全不是可选项

🔑 JWT 安全

  • Secret 从环境变量注入，禁止硬编码
  • Access Token 有效期 ≤ 60 分钟
  • Refresh Token 7 天，支持吊销
  • 使用 HTTPS 传输 Token

🛡️ CSRF / XSS

  • CSRF 保护在 Web 表单场景必须开启
  • API 场景使用 JWT 替代 CSRF
  • 模板引擎默认转义 HTML
  • Content-Security-Policy Header

🗄️ SQL 注入防护

  • 使用 V ORM 的参数化查询
  • 禁止拼接 SQL 字符串
  • Repository 方法自动参数化
  • 输入校验在 Controller 层完成

📋 日志脱敏

  • 密码、Token 自动脱敏
  • 信用卡号只显示后四位
  • PII 数据按合规要求处理
  • 生产日志级别 ≥ info

// 安全配置示例 jwt_config := security.JwtConfig{ secret: os.get_env('JWT_SECRET')! // 环境变量 issuer: 'my-app' expiration_minutes: 15 // 短有效期 } csrf_config := security.CsrfConfig{ enabled: true cookie_secure: true // HTTPS only cookie_same_site: 'Strict' }

性能优化
编译期优化 + 运行期调优

🏊 连接池

  • min_size = 常态并发量
  • max_size ≤ 数据库最大连接数
  • 设置 idle_timeout 释放空闲连接
  • 监控 pool.stats() 的 wait_count

💨 缓存策略

  • 热点数据使用 Redis 缓存
  • Singleflight 防缓存击穿
  • 缓存键使用 namespace:key
  • 设置合理 TTL，避免数据不一致

🔒 锁优化

  • 单进程用 LocalMutex，跨节点用 DistributedLock
  • 锁粒度越细越好
  • 必须设置 TTL 防死锁
  • 使用 defer { lock.release() }

📋 日志优化

  • 生产使用 JSON 编码器
  • 日志级别 ≥ info
  • 热路径避免 debug 字符串拼接
  • 使用 is_enabled(.debug) 前置检查
