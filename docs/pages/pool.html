import photon.pool // 1. 定义池化对象（必须实现 PooledObject 接口） struct DbConnection { pub mut: host string is_valid bool = true } pub fn (mut c DbConnection) close() ! { // 关闭连接 } pub fn (c &DbConnection) is_valid() bool { return c.is_valid } // 2. 创建连接池 mut p := pool.new_pool_with_config('db-pool', fn () !voidptr { // 工厂函数 — 创建新连接 conn := &DbConnection{host: 'localhost'} return unsafe { conn } }, 2, 10) // min=2, max=10 // 3. 初始化（预创建 min_size 个对象） p.initialize()! // 4. 获取对象 obj := p.acquire()! conn := unsafe { &DbConnection(obj) } // ... 使用连接 // 5. 归还对象 p.release(obj) // 6. 查看池状态 stats := p.stats() println('Active: ${stats.active}, Idle: ${stats.idle}') // 7. 关闭池 p.close()!

配置参数
min_size / max_size / idle_timeout

max_size10池中最多允许的对象数
idle_timeout_seconds300空闲对象超时时间（秒）
name—池名称（日志标识）
factory—对象工厂函数

健康检查
自动剔除无效连接
v
// 获取对象时自动检查有效性 obj := p.acquire()! // 内部调用 obj.is_valid() // 手动清理无效对象 p.cleanup_invalid() // HikariCP 等价：connectionTimeout / validationTimeout
🚀 生产实践
  • min_size 设置为常态并发量，减少冷启动时的连接创建开销
  • max_size 不超过数据库最大连接数 / 系统文件描述符上限
  • 设置 idle_timeout 避免空闲连接长期占用资源
  • 使用 defer { pool.release(conn) } 确保连接一定归还
  • 监控 pool.stats()，当 wait_count 持续增长时扩容
