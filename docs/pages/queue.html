Job 定义
Job 接口 · job_type / handle / tries / backoff
v
import photon.queue // 1. 定义 Job（必须实现 Job 接口） pub struct SendWelcomeEmail { pub: user_id int email string } pub fn (j &SendWelcomeEmail) job_type() string { return 'send_welcome_email' } pub fn (mut j SendWelcomeEmail) handle() ! { // 实际执行逻辑 send_email(j.email, 'Welcome!')! } pub fn (j &SendWelcomeEmail) tries() int { return 3 // 最多重试 3 次 } pub fn (j &SendWelcomeEmail) backoff() []i64 { return [10, 30, 60] // 第 1 次重试等 10s，第 2 次 30s，第 3 次 60s } // Laravel 等价：implements ShouldQueue // Spring 等价：@Async + @Retryable

Job 调度
即时 / 延迟 / 链式 / 批量
v
mut dispatcher := queue.new_dispatcher() // 即时调度 dispatcher.dispatch(SendWelcomeEmail{user_id: 42, email: 'alice@example.com'})! // 延迟调度（60 秒后执行） dispatcher.dispatch_delayed( SendWelcomeEmail{user_id: 42, email: 'bob@example.com'}, delay_secs: 60 )! // 链式调度（按顺序执行，前一个成功才执行下一个） dispatcher.dispatch_chain([ ValidateOrder{order_id: 'ORD-001'}, ProcessPayment{order_id: 'ORD-001'}, SendConfirmation{order_id: 'ORD-001'}, ])! // 批量调度 dispatcher.dispatch_batch([ SendWelcomeEmail{user_id: 1, email: 'a@example.com'}, SendWelcomeEmail{user_id: 2, email: 'b@example.com'}, SendWelcomeEmail{user_id: 3, email: 'c@example.com'}, ])! // Laravel 等价：dispatch() / dispatch(new Job)->delay(60) / Bus::chain([])
👷
Worker
后台轮询 + 执行 + 重试
v
// 注册 Job 处理器 mut worker := queue.new_worker(queue_backend) worker.register('send_welcome_email', SendWelcomeEmail{}) // 启动 Worker（轮询 + 执行） worker.run() // 阻塞运行 // 优雅停止 worker.stop() // Laravel 等价：php artisan queue:work
🚀 生产实践
  • Job 必须是幂等的（重试不会产生副作用）
  • 设置合理的 tries() 和 backoff()，避免无限重试
  • 长时间运行的任务拆分为链式 Job，每步可独立重试
  • Worker 进程使用 systemd/supervisor 管理，崩溃自动重启
  • 监控队列积压（pending jobs），及时扩容 Worker
  • 使用 @[job] + scan_job[T]() 实现编译期 Job 元数据扫描，避免手动配置
  • 使用 WorkerConfig 和 WorkerLifecycle 精细控制 Worker 行为和生命周期
🏷️
Job 注解与编译期扫描
@[job] · @[retry] · @[backoff] · @[timeout] · scan_job[T]()

使用注解声明式定义 Job 元数据，通过编译期扫描自动提取，避免手动配置。

注解 / 方法说明
@[job]标记 struct 为 Job Bean
@[job: 'queue_name']指定目标队列
@[retry(3)]最大重试次数
@[backoff('1000,5000,10000')]退避策略（毫秒）
@[timeout(30)]Job 超时时间（秒）
scan_job[T]()编译期扫描 Job struct 属性，返回 JobMetadata
register_job[T]()将 Job 类型注册到 Dispatcher
v
import photon.queue // 使用注解定义 Job @[job] @[retry(5)] @[backoff('1000,5000,30000')] @[timeout(60)] pub struct ProcessPayment { pub: order_id string amount f64 } pub fn (j &ProcessPayment) job_type() string { return 'process_payment' } pub fn (mut j ProcessPayment) handle() ! { // 支付处理逻辑 } pub fn (j &ProcessPayment) tries() int { return 5 } pub fn (j &ProcessPayment) backoff() []i64 { return [1000, 5000, 30000] } // 编译期扫描 meta := queue.scan_job[ProcessPayment]() println(meta.job_type) // 'ProcessPayment' println(meta.max_retries) // 5 println(meta.timeout_secs) // 60 // 自动注册到分发器 mut dispatcher := queue.new_dispatcher() queue.register_job[ProcessPayment](mut dispatcher)! // Spring 等价：@Async + @Retryable // Laravel 等价：implements ShouldQueue

高级分发
指定队列 · 批量 · 链式延迟 · 统计 · 暂停/恢复
方法说明
dispatch_on(queue_name, job)分发到指定队列
dispatch_bulk(jobs)批量分发
dispatch_chain_with_delay(jobs, delay)链式延迟分发（Job 间有延迟）
QueueStats队列统计（pending / completed / failed / total）
pause(queue_name) / resume(queue_name)暂停/恢复队列
v
mut dispatcher := queue.new_dispatcher() // 分发到指定队列 dispatcher.dispatch_on('payments', ProcessPayment{order_id: 'ORD-001', amount: 99.9})! // 批量分发 dispatcher.dispatch_bulk([ SendWelcomeEmail{user_id: 1, email: 'a@example.com'}, SendWelcomeEmail{user_id: 2, email: 'b@example.com'}, SendWelcomeEmail{user_id: 3, email: 'c@example.com'}, ])! // 链式延迟分发 dispatcher.dispatch_chain_with_delay([ ValidateOrder{order_id: 'ORD-001'}, ProcessPayment{order_id: 'ORD-001'}, SendConfirmation{order_id: 'ORD-001'}, ], delay_secs: 5)! // 队列统计 stats := dispatcher.stats('default') println('Pending: ${stats.pending}, Completed: ${stats.completed}, Failed: ${stats.failed}') // 暂停/恢复 dispatcher.pause('payments') dispatcher.resume('payments')

Worker 生命周期管理
WorkerConfig · WorkerLifecycle · drain · 优雅关闭
类型 / 方法说明
WorkerConfig配置 Worker 行为（poll_interval / max_jobs / concurrency）
WorkerLifecycleWorker 生命周期钩子（on_start / on_stop / on_error）
drain()排空剩余 Job 后停止
job_count()当前待处理 Job 数量
is_idle()检查 Worker 是否空闲
v
import photon.queue // 配置 Worker config := queue.WorkerConfig{ poll_interval: 100 // ms max_jobs: 1000 concurrency: 4 } // Worker 生命周期钩子 lifecycle := queue.WorkerLifecycle{ on_start: fn () { println('Worker 已启动') } on_stop: fn () { println('Worker 已停止') } on_error: fn (err IError) { eprintln('Worker 错误: ${err}') } } mut worker := queue.new_worker_with_config(queue_backend, config, lifecycle) worker.register('process_payment', ProcessPayment{}) // 检查 Worker 状态 println('Pending jobs: ${worker.job_count()}') println('Is idle: ${worker.is_idle()}') // 优雅关闭：处理完剩余任务后停止 worker.drain()
