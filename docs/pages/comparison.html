关键差异
Photon 的独特优势

⚡ 编译期 DI vs 运行期反射

Spring 使用运行期反射扫描 Bean，Laravel 使用运行期容器解析。Photon 通过 comptime $for 在编译期完成所有依赖解析，零运行时开销。

🔒 编译期循环依赖检测

Spring 在运行时通过三级缓存解决循环依赖（有坑），Laravel 允许循环依赖。Photon 在编译期直接报错，强制你使用 DeferredProvider 打破循环。

🚀 单二进制部署

V 编译为原生机器码，Photon 应用编译后是一个独立二进制文件，无需 JVM / PHP Runtime / Docker，直接部署。

📦 极小内存占用

无 JVM 堆开销（Spring），无 PHP-FPM 进程池（Laravel）。Photon 应用内存占用通常在 10-30 MB，适合嵌入式和边缘计算。
