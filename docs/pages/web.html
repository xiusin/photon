Web · Web 框架

注解驱动路由、可组合中间件链、统一响应封装、CORS 支持 — Spring MVC 的 V 语言实现，基于 veb 高性能 HTTP 引擎。深度集成 veb 16 项原生能力：预压缩、CSRF HMAC、WebSocket、SSE、Host 路由、翻译桥接等。

⚡
veb 深度集成
16 项原生能力桥接 — 预压缩、CSRF、WebSocket、Host 路由、翻译、资源管理

Photon Web 模块深度桥接了 veb 框架的 16 项原生能力，在保持 Photon API 风格的同时，让开发者可以直接使用 veb 的高性能特性。所有桥接均通过 Photon 的 MiddlewareChain 或 WebModule 体系暴露，无需直接操作 veb 底层 API。

💡 设计原则：编译期优于运行期，显式优于隐式。所有 veb 桥接功能均为可选引入，不影响现有代码。

静态文件预压缩与即时压缩

桥接 veb.Context.serve_precompressed_file() 和 veb.Context.serve_compressed_static()。当客户端发送 Accept-Encoding: gzip 或 zstd 时，自动优先使用预压缩的 .gz / .zst 文件（零拷贝），未找到则即时压缩并缓存到磁盘。

v
import photon.web fn main() { mut wm := web.init_web_module() // 1. 注册静态资源目录 wm.add_resource_handler('/static/**', './public') // 2. 启用自动压缩（zstd 优先，gzip 回退） wm.enable_static_compression( enable: true max_size: 2_000_000 // 2MB 以下自动压缩 ) // 3. 仅 gzip 模式（兼容性更好） // wm.enable_static_compression(enable_gzip_only: true) // 4. 仅 zstd 模式（更好的压缩比） // wm.enable_static_compression(enable_zstd_only: true) } // 压缩策略： // 1. 查找 .zst / .gz 预压缩文件（零拷贝，最快） // 2. 未找到 → 即时压缩并写入 .zst / .gz 缓存 // 3. 客户端不支持压缩 → 返回原始文件

Markdown 内容协商

桥接 veb.StaticHandler.enable_markdown_negotiation。当客户端发送 Accept: text/markdown 时，自动返回 .md 变体文件。

v
// 启用 Markdown 内容协商 wm.enable_markdown_negotiation() // 请求 /docs/api → 按优先级查找： // 1. ./public/api.md // 2. ./public/api.html.md // 3. ./public/api/index.html.md // 4. 回退到 ./public/api（原始文件） // 客户端请求示例： // curl -H "Accept: text/markdown" https://app.com/docs/api

CSRF 防护（HMAC-SHA256 签名）

桥接 veb.csrf.CsrfConfig。相比 security/csrf.v 的 Double-Submit Cookie 模式，此实现提供 HMAC-SHA256 签名的 CSRF token、Origin/Referer 双重验证、Token 过期时间嵌入。

v
import photon.web // 1. 创建 CSRF 配置 csrf := web.new_csrf_bridge( secret: 'your-hmac-secret-key' allowed_hosts: ['example.com', 'app.example.com'] max_age: 86400 // 24 小时 check_origin_and_referer: true // 双重验证 same_site: .same_site_strict_mode ) // 2. 注册为前置中间件 wm.use(csrf.to_before_middleware()) // 3. 在表单页面生成 CSRF Token @[get('/form')] pub fn (mut ctx Context) show_form() veb.Result { token := web.csrf_set_token(mut ctx, &csrf) html := '<form method="POST">${web.csrf_token_input(token, &csrf)}<input...></form>' return ctx.html(html) } // 验证流程： // 1. GET/HEAD/OPTIONS 直接放行 // 2. POST/PUT/DELETE → 检查 Origin + Referer // 3. 从表单/头提取 CSRF Token // 4. 验证 Token 的 HMAC 签名和过期时间 // 5. 验证 Cookie 中的 HMAC 与 Token 匹配

WebSocket 长连接支持

桥接 veb.Context.takeover_conn() + RFC 6455 握手。通过接管 TCP 连接实现 WebSocket 协议升级，支持文本帧读写。

v
import photon.web import veb // 注册 WebSocket 路由 wm.get('/ws', fn (ctx_ptr voidptr, _ map[string]string) veb.Result { mut ctx := unsafe { &veb.Context(ctx_ptr) } // 1. 执行 WebSocket 升级握手 mut ws := web.upgrade_websocket(mut ctx, 'secret') or { return ctx.text('WebSocket upgrade failed') } // 2. 循环读取消息 for { msg := ws.read() or { break } // 3. 回显消息 ws.send('echo: ${msg}')! } // 4. 关闭连接 ws.close() return veb.Result{} })

Host 级路由隔离

桥接 veb.controller_host()。通过 host 参数实现多租户/多域名场景下的路由隔离。

v
// 1. 注册带 host 限制的路由 wm.router.register_with_host('GET', '/admin', admin_handler, 'admin.example.com') wm.router.register_with_host('GET', '/admin', tenant_handler, 'tenant.example.com') // 2. 带 host + 中间件的路由 wm.router.register_with_middleware( 'GET', '/api/internal', internal_handler, [auth_middleware], [], 'internal.corp.com' ) // 路由匹配规则： // - route.host 为空 → 匹配任意 host // - route.host 非空 → 必须与请求 Host 头完全匹配 // - host 限制的路由优先于无限制路由

桥接 veb.request_id.Config。相比固定的 request_id_middleware，此配置支持自定义生成器、请求头名称、强制生成等选项。

v
// 自定义 Request ID 配置 config := web.RequestIdConfig{ header: 'X-Correlation-ID' // 自定义头名 force: true // 总是生成新 ID } wm.use(web.configurable_request_id_middleware(config)) // 带自定义生成器 + 跳过函数 config2 := web.RequestIdConfig{ header: 'X-Request-ID' generator: fn () string { return 'req-${time.ticks().hex()}' } next: fn (ctx &veb.Context) bool { return ctx.req.url.starts_with('/health') // 跳过健康检查 } } wm.use(web.configurable_request_id_middleware(config2))

控制器挂载
mount_controller · mount_controller_auto — 编译期 comptime 路由扫描

Photon Web 的核心能力：通过 comptime $for 扫描控制器 T 的方法，自动发现路由注解（@[get]、@[post] 等）并注册为 RouteHandler 闭包。控制器可在任意目录、任意包中定义，只需在挂载点 import 即可。

路由注解格式

支持两种写法，推荐使用合并式（Spring 风格）：

格式写法说明
合并式（推荐）@[get: '/users']Spring 风格，方法与路径合并在一个注解中
分离式@[get] + @['/users']veb 风格，HTTP 方法和路径分开标注

控制器前缀

v
// 使用 @[prefix] 注解定义路由前缀 @[controller] @[prefix: '/api/v1'] pub struct UserController { web.BaseController veb.Context user_service &UserService @[autowired] } @[get: '/users'] pub fn (mut c UserController) list(mut ctx veb.Context, params map[string]string) veb.Result { return ctx.text('{"users":[]}') } @[get: '/users/:id'] pub fn (mut c UserController) show(mut ctx veb.Context, params map[string]string) veb.Result { id := params['id'] or { '' } return ctx.text('{"id":"${id}"}') }

挂载 API

方法说明
rr.mount_controller[T](controller, prefix)扫描 T 的路由方法并注册到 RouteRegistry
rr.mount_controller_auto[T](controller, default_prefix)自动从 @[prefix] 注解提取前缀，无注解则使用 default_prefix
wm.mount_controller[T](controller, prefix)通过 WebModule 挂载（委托给 RouteRegistry）
wm.mount_controller_auto[T](controller, default_prefix)通过 WebModule 自动挂载
core.wire_controller[T]()创建控制器 + 自动注入 @[autowired] 依赖

完整挂载示例

v
import photon.core import photon.web import veb // 主应用结构体 pub struct App { web.WebModule } // 1. 定义控制器（可在任意文件/包中） @[controller] @[prefix: '/api/v1'] pub struct UserController { user_service &UserService @[autowired] } @[get: '/users'] pub fn (mut c UserController) list(mut ctx veb.Context, params map[string]string) veb.Result { return ctx.text('{"users":[]}') } // 2. 在 main 中挂载 fn main() { mut app := core.new_application_context() app.refresh()! // 创建应用实例 mut server := &App{ WebModule: web.init_web_module() } // 创建控制器（自动注入依赖）+ 挂载路由 ctrl := app.wire_controller[UserController]()! server.WebModule.mount_controller[UserController](ctrl, '/api/v1') // 或使用 mount_controller_auto（自动提取 @[prefix]） // server.WebModule.mount_controller_auto[UserController](ctrl, '/api') // 3. 在 before_request 中分发请求 // pub fn (mut ctx Context) before_request() { // ctx.app.WebModule.handle_request(mut ctx) // } }
💡 控制器方法签名约定：路由处理方法必须具有签名 fn (mut c T, mut ctx veb.Context, params map[string]string) veb.Result。mount_controller[T]() 通过 $if method.return_type is veb.Result 确保只对返回 veb.Result 的方法生成调用代码，非路由方法（构造器等）不会被匹配。
⚠️ V 0.5.1 comptime 限制：闭包内不能直接使用 $method，因此通过 dispatch_controller_method[T]() 顶层泛型函数进行分发。$for 循环中不允许 continue，使用 if 嵌套代替。
🚀 生产实践
  • 使用 recovery_middleware 防止 panic 导致服务崩溃
  • request_id_middleware + Logger MDC 实现请求级日志追踪
  • CORS 配置不要使用 *，明确指定允许的 Origin
  • 中间件顺序很重要：recovery → logging → cors → auth → business
  • 路径参数使用 :id 风格，避免路由冲突
  • 使用 @[prefix: '/api/v1'] + mount_controller_auto[T]() 实现自动化路由挂载
  • 控制器拆分到不同文件/包中，通过 wire_controller[T]() 注入依赖后挂载
  • 路由注解推荐使用合并式 @[get: '/users']，与 Spring @GetMapping 风格一致
  • 静态资源启用 enable_static_compression，预压缩 .gz/.zst 文件可减少 70% 传输量
  • 多租户场景使用 register_with_host 实现 Host 级路由隔离
  • WebSocket / SSE 长连接路由放在中间件链末尾，避免被 recovery_middleware 超时中断
  • 生产环境使用 veb.RunParams{family: .ip} 启用 fasthttp 后端，QPS 提升 2-3 倍
  • 前端资源使用 AssetManager 统一管理，开启 minify + combine 减少请求数
