编码器
JSON / Console 双格式
v
// JSON 编码器 — 生产环境推荐（便于 ELK/Grafana 解析） mut json_enc := logger.new_json_encoder(logger.EncoderConfig{ time_key: 'ts' level_key: 'level' message_key: 'msg' time_format: .rfc3339 }) log.set_encoder(json_enc) // 输出示例： // {"ts":"2025-01-15T10:30:00Z","level":"info","msg":"Server started"} // Console 编码器 — 开发环境（可读性强） mut console_enc := logger.new_console_encoder(logger.EncoderConfig{ time_format: .iso8601 }) log.set_encoder(console_enc) // 输出示例： // 2025-01-15 10:30:00 [INFO] Server started
🏷️
MDC 上下文
Mapped Diagnostic Context — 请求级追踪

MDC 允许在请求上下文中注入键值对，后续所有日志自动携带这些字段，非常适合请求追踪。

v
// 在中间件中注入 Request-ID log.put('request_id', 'req-abc123') log.put('user_id', '42') // 后续所有日志自动携带 request_id 和 user_id log.info('Processing order') // {"ts":"...","level":"info","msg":"Processing order","request_id":"req-abc123","user_id":"42"} // 请求结束后清理 log.remove('request_id') log.remove('user_id') // Spring 等价：MDC.put("requestId", "req-abc123") // Go Zap 等价：logger.With(zap.String("request_id", "req-abc123"))

🚀 生产实践
  • 生产环境使用 JSON 编码器 + RFC3339 时间，便于 ELK/Loki 采集分析
  • 通过 request_id_middleware 自动注入 Request-ID 到 MDC，实现全链路追踪
  • 敏感信息（密码、Token）在日志中自动脱敏
  • 设置合理的日志级别：开发 debug，生产 info
  • 热路径避免 debug 级别的字符串拼接，先检查 log.is_enabled(.debug)
